# 01 — 환경 준비와 로그인

## 1) 작업 폴더로 이동

> 설치 스크립트가 작업 폴더를 이미 생성해 두었습니다. 위치는 고정입니다.
> - Windows: `C:\Users\계정명\copilot-workspace\invaders`
> - Mac: `~/copilot-workspace/invaders`
>
> 따라서 새로 만들 필요 없이 해당 폴더로 이동만 하면 됩니다. 설치를 하지 않았다면 [설치 가이드](../setup/설치_가이드.md)를 먼저 진행하십시오.

터미널(Windows는 PowerShell, Mac은 터미널)에서 작업 폴더로 이동합니다.

- Windows (PowerShell):
  ```powershell
  cd "$env:USERPROFILE\copilot-workspace\invaders"
  ```
- Mac (터미널):
  ```bash
  cd ~/copilot-workspace/invaders
  ```

### 이 작업 폴더의 의미

작업 폴더(`copilot-workspace\invaders`)는 단순한 빈 폴더가 아니라 **작업 공간(workspace)** 입니다. 세 가지 의미가 있습니다.

1. **Copilot의 작업 기준점입니다.** Copilot은 현재 위치한 폴더를 기준으로 동작합니다. 게임 파일이 여기에 생성되고, 이후 만들 **지침·스킬** 도 이 폴더 안(`.github/` 폴더)에 저장되어 이 프로젝트에만 적용됩니다. 즉 폴더가 곧 프로젝트의 범위입니다.

2. **위치가 고정되어 있어 파일을 잃지 않습니다.** 어느 위치에서 터미널을 열든 작업 폴더는 항상 `copilot-workspace\invaders` 한 곳입니다. 또한 이후 Copilot으로 다른 작업을 할 때도 이 `copilot-workspace` 폴더를 계속 사용하면 됩니다. 상설 작업 공간으로 쓰기 위한 구조입니다.

3. **변경 이력 관리(git)가 켜져 있습니다.** 설치 스크립트가 이 폴더에 git을 초기화해 두었습니다. Copilot이 변경한 내용을 이전 상태로 되돌릴 수 있고, 폴더 전체를 GitHub에 올리면 다른 사람도 동일한 환경을 그대로 받을 수 있습니다(보너스 모듈에서 실습).

## 2) 갤러리 주소 등록

생성한 게임을 진행자의 공용 갤러리로 전송하려면 주소를 먼저 등록합니다. **Copilot 실행 전에** 입력해야 합니다.

- Windows (PowerShell):
  ```powershell
  $env:LAB502_DASHBOARD_URL = "https://rg-hub-web-0614.azurewebsites.net"
  ```
- Mac (터미널):
  ```bash
  export LAB502_DASHBOARD_URL="https://rg-hub-web-0614.azurewebsites.net"
  ```

> 이 주소는 새 터미널 창을 열 때마다 다시 입력해야 합니다.

## 3) Copilot 실행

```bash
copilot
```

처음 실행하면 몇 가지를 확인합니다.

- **이 폴더를 신뢰하는가** — "Yes, and remember this folder for future sessions"(숫자 **2**)를 선택합니다.
- **여러 줄 입력을 지원할 것인가** — **Yes** 를 선택합니다. 이후 `Shift+Enter` 로 긴 문장을 여러 줄에 걸쳐 입력할 수 있습니다.

> Copilot CLI에서는 모든 선택지를 해당 숫자 키로 바로 고를 수 있습니다.

## 4) GitHub 로그인

실행된 대화창에서 다음을 입력합니다.

```
/login
```

안내에 따라 브라우저에서 GitHub 인증을 완료하면 준비가 끝납니다. 이전 세션에서 이미 로그인했다면 이 단계는 생략됩니다.

인증이 완료되면 GitHub MCP 서버에도 자동으로 연결됩니다.

대화창에서 사용할 수 있는 입력 방식입니다.

| 입력 | 의미 | 예시 |

# 스크린샷 Lab 502 Community Hub 업로드 방법

우주 침략자 게임 스크린샷을 Lab 502 Community Hub에 업로드한 절차 정리.

## 1. 사전 준비

- **대시보드 URL (환경 변수)**: 업로드 대상 서버 주소를 환경 변수로 지정한다.

```powershell
$env:LAB502_DASHBOARD_URL = "https://rg-hub-web-0614.azurewebsites.net"
```

- **업로드 엔드포인트**: `<DASHBOARD_URL>/api/image`
- **HTTP 메서드**: `POST`
- **전송 형식**: `multipart/form-data`, 필드 이름은 `image`

## 2. 캡처

Playwright로 게임 페이지를 연 뒤 게임을 시작(`게임 시작` 버튼 클릭)하고 진행 중 화면을 PNG로 저장.

- 저장 위치 예: `C:\Users\<user>\AppData\Local\Temp\screenshot_<timestamp>.png`

## 3. 업로드 (성공한 방법 - curl 멀티파트)

PowerShell 스크립트(`share-screenshot.ps1`)의 `Invoke-WebRequest` 방식은 응답 파싱 단계에서
`.NET` 예외("Object reference not set to an instance of an object")가 발생했다.
대신 `curl.exe`로 멀티파트 업로드하여 정상 처리(HTTP 200)했다.

```powershell
$env:LAB502_DASHBOARD_URL = "https://rg-hub-web-0614.azurewebsites.net"
$img = "C:\Users\<user>\AppData\Local\Temp\screenshot_<timestamp>.png"

curl.exe -s -o - -w "`nHTTP_STATUS:%{http_code}`n" `
  -X POST "$env:LAB502_DASHBOARD_URL/api/image" `
  -F "image=@$img;type=image/png"
```

### 핵심 옵션 설명

| 옵션 | 의미 |
|------|------|
| `-X POST` | POST 요청 |
| `-F "image=@<path>;type=image/png"` | `image` 필드로 파일을 멀티파트 전송, MIME 타입 명시 |
| `-w "HTTP_STATUS:%{http_code}"` | 응답 HTTP 상태 코드 출력 |
| `-s` | 진행률 표시 숨김 |

## 4. 성공 확인

- 응답: `HTTP_STATUS:200`
- 서버 저장 파일명 예: `432ee4f2-cf33-4118-950e-2c9bd3991621.png`

`HTTP_STATUS:200` 과 저장 파일명이 반환되면 업로드 성공으로 판단한다.

## 5. 트러블슈팅

| 증상 | 원인 | 조치 |
|------|------|------|
| `Unable to connect to the remote server` | `LAB502_DASHBOARD_URL` 미설정(로컬 기본값) | 올바른 대시보드 URL 환경 변수 설정 |
| `Object reference not set to an instance of an object` | `Invoke-WebRequest` 응답 파싱 문제 | `curl.exe` 멀티파트 방식으로 대체 |
