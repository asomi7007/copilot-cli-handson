# 02 — 플러그인 설치

## 플러그인이란

플러그인은 GitHub Copilot의 기능을 확장하는 배포 가능한 패키지입니다. 스킬, 커스텀 에이전트, 훅, MCP 서버 설정, 스크립트 등 여러 구성 요소를 하나로 묶어, 마켓플레이스에서 발견하고 설치할 수 있게 합니다. 잘 구성된 워크플로우를 공유하기 쉽게 만들면서도 버전과 설정은 사용자가 통제할 수 있습니다.

이 실습에서는 게임 화면을 캡처해 공유하는 도구가 포함된 플러그인을 설치합니다.

## 설치 절차

Copilot이 실행된 대화 상태에서 다음을 순서대로 입력합니다.

1. 현재 등록된 마켓플레이스 목록 확인:
   ```
   /plugin marketplace list
   ```
   기본 마켓플레이스인 **github/copilot-cli-plugins** 와 **github/awesome-copilot** 가 보입니다.

2. 이번 실습용 마켓플레이스 등록:
   ```
   /plugin marketplace add microsoft/Build26-LAB502-make-github-copilot-work-your-way-custom-tools-context-and-workflows
   ```

3. 해당 마켓플레이스의 플러그인 목록 확인:
   ```
   /plugin marketplace browse community-hub
   ```
   **space-invaders-makers** 플러그인 하나가 표시됩니다.

4. 설치:
   ```
   /plugin install space-invaders-makers@community-hub
   ```

5. 설치 확인:
   ```
   /plugin list
   ```
   목록에 **space-invaders-makers@community-hub** 가 있으면 정상입니다.

설치되는 구성: **web-screenshotter 에이전트**, **share-screenshot 스킬**, 그리고 활동 기록용 **훅 4개**입니다.

> **보안 참고:** 플러그인의 스킬은 사용자의 권한으로 코드를 실행할 수 있습니다. 두 가지를 유의하십시오. 번들된 스크립트는 셸 권한으로 실행되며, `SKILL.md` 는 에이전트가 읽으므로 의도치 않은 방향으로 동작을 유도할 수 있습니다. 신뢰할 수 있는 출처(여기서는 Microsoft 공식)의 플러그인만 설치하고, 가능하면 설치 전에 `SKILL.md` 와 스크립트를 확인하십시오.
