# 08 — 정리

빈 폴더에서 출발해 게임을 생성하고, 공용 갤러리에 공유하고, 지침과 스킬을 직접 작성했습니다.

## 수행한 작업

- **환경 준비** — Copilot CLI 설치·로그인, 작업 공간 폴더 구성
- **플러그인 설치** — 스킬·에이전트·훅을 한 패키지로 설치
- **게임 생성** — 계획(Plan) 모드로 설계 후 단일 HTML 파일 게임 생성
- **게임 공유** — 캡처·업로드 전 과정을 커스텀 에이전트에 위임
- **구성 요소 확인** — Copilot을 구성하는 9가지 요소 정리
- **지침 작성** — 항상 적용되는 저장소 규칙 생성
- **스킬 작성** — 의도로 호출되는 여러 단계 워크플로우 생성

## 9가지 구성 요소 요약

| 정식 명칭 | 작동 방식 | 적합한 용도 |
|----------|----------|------------|
| **지침 (Instructions)** | 저장소·사용자 단위로 매 세션 자동 로드 | 저장소 전반의 규칙, 규약, 아키텍처 결정 |
| **프롬프트 파일 (Prompt files)** | `/명령` 으로 명시적 호출 | 매개변수가 있는 반복 워크플로우 |
| **스킬 (Skills)** | 설명과 의도가 일치하면 자동 호출 | 스크립트·참고자료를 포함한 작업 단위 능력 |
| **도구 (Tools)** | 작업 수행에 필요할 때 호출 | 검색·읽기·편집·명령 실행 |
| **커스텀 에이전트 (Custom agents)** | 선택 또는 위임 | 전용 도구·모델·가드레일을 갖춘 세션 단위 보조자 |
| **서브에이전트 (Subagents)** | Copilot이 위임이 유리하다고 판단 시 | 집중 작업 격리, 메인 맥락 정리, 병렬 탐색 |
| **훅 (Hooks)** | 라이프사이클 이벤트 발생 시 자동 | 포매팅·린팅·감사 로깅, 맥락 주입, 도구 차단 |
| **MCP 서버** | 설정 시 도구가 노출됨 | 데이터베이스·API·내부 서비스 연결 |
| **플러그인 / 워크플로우** | 단위로 설치 / 이벤트로 트리거 | 구성 요소 묶음 배포 / 저장소 자동화 |

이 요소들은 상호 보완적으로 계층을 이룹니다. 지침이 기반을 깔고 → 프롬프트·에이전트가 워크플로우를 패키징하고 → 스킬이 능력을 더하고 → 도구가 실제로 실행하고 → 서브에이전트가 복잡한 작업을 분리하고 → 훅이 자동화하고 → MCP가 외부로 확장하고 → 플러그인이 전체를 묶어 공유합니다.

## 자주 헷갈리는 셋: 스킬 vs 지침 vs MCP

실제로 가장 많이 헷갈리는 것이 **스킬·지침·MCP**입니다. "언제 무엇을 쓰는가"를 한 줄로 구분하면 다음과 같습니다.

| 구분 | 무엇인가 | 언제 쓰나 (예) |
|------|---------|---------------|
| **스킬 (Skills)** | 특정 작업을 **항상 정해진 절차대로** 수행하게 하는 실행 매뉴얼 | "게임 업로드", "스크린샷 공유"처럼 **반복되는 다단계 작업**의 절차를 고정 |
| **지침 (Instructions)** | 프로젝트 전반에 **상시 적용**되는 스타일·제약 규칙 | "단일 HTML 파일 유지", "외부 라이브러리 금지"처럼 **항상 지켜야 할 규칙** |
| **MCP 서버** | 외부 서비스·DB·API와의 **실시간 데이터 연동** 통로 | 갤러리 목록 조회, 외부 시스템 질의처럼 **바깥 데이터·기능이 필요할 때** |

기억할 점: **스킬은 "어떻게 할지(절차)", 지침은 "항상 지킬 것(규칙)", MCP는 "바깥과 어떻게 연결할지(연동)"** 를 담당합니다. 셋은 경쟁이 아니라 역할 분담입니다.

> 보조 기준: 스킬·지침은 모두 사람이 읽고 고치는 **문서**이고, 자동으로 로드됩니다(지침은 매 세션, 스킬은 의도가 맞을 때). MCP는 문서가 아니라 **연결 설정**이라, 연결해 두면 그때부터 새로운 도구가 생깁니다.

## 실제 업무 적용

이 실습에서 다룬 방식은 게임에 국한되지 않습니다. 동일한 방법을 다음에 적용할 수 있습니다.

- **파일 정리** — "이 폴더의 사진을 날짜별 폴더로 정리해줘"
- **문서 작성** — "이 회의록을 요약해 보고서로 작성해줘"
- **자료 조사** — "이 주제의 최신 정보를 찾아 표로 정리해줘"
- **반복 업무 자동화** — 자주 하는 작업을 지침·스킬·프롬프트 파일로 만들어 재사용

모두 터미널에서 자연어로 지시하며, 자주 쓰는 작업은 구성 요소로 만들어 두면 이후 작업이 빨라집니다.
