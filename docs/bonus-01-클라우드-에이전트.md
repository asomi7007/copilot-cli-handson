# 보너스 1 — 클라우드 에이전트 (Copilot cloud agent)

이 실습에서 만든 지침·스킬 같은 저장소 커스터마이징 파일은 로컬에서만 동작하는 것이 아닙니다. 동일한 파일이 **Copilot 클라우드 에이전트** — GitHub가 호스팅하는 에이전트로, 저장소를 조사하고 계획을 세우고 브랜치에서 변경한 뒤 작업을 넘기거나 PR을 여는 — 에서도 그대로 동작합니다. 이 모듈에서는 프로젝트를 GitHub에 올리고, 작업을 클라우드 에이전트에 맡겨, 이미 작성한 지침을 존중하는 PR이 생성되는 것을 확인합니다.

## 클라우드 에이전트란

클라우드 에이전트는 보안이 적용된 클라우드 환경에서 동작하며 개발 작업을 백그라운드로 수행하는 AI 개발자입니다. 작업을 시작하는 방식에 따라 저장소 조사, 계획 제안, 브랜치에서의 변경, 검사 실행을 거쳐 PR을 생성합니다.

이번에는 이슈 기반 워크플로우를 사용합니다. GitHub 이슈를 Copilot에 할당하면 다음을 수행합니다.

- 이슈 설명과 연결된 맥락을 읽는다.
- 저장소를 복제하고 브랜치를 체크아웃한다.
- 파일을 읽고 코드를 작성하고 명령을 실행하며 자율적으로 변경을 구현한다.
- 결과를 담은 PR을 연다.

저장소에서 직접 동작하므로, 로컬 세션과 동일한 `.github/copilot-instructions.md` 를 그대로 사용합니다.

> 클라우드 에이전트는 GitHub Actions 기반의 호스팅 환경에서 실행됩니다. git·node·curl 등 표준 도구에 접근할 수 있지만, 사용자의 로컬 머신이나 로컬에 설치된 MCP 서버에는 접근하지 못합니다. 저장소에 커밋된 커스터마이징 파일만 사용합니다.

## 1) 프로젝트를 GitHub에 올리기

> 이 단계에는 GitHub 계정과 git(또는 gh)이 필요합니다.

작업 폴더(`invaders`)에서 Copilot에 자연어로 요청할 수 있습니다.

```
이 폴더를 내 GitHub에 새 저장소로 올려줘. 저장소 이름은 space-invaders-내이름 으로 해줘.
```

Copilot이 git·gh 명령으로 저장소 생성과 업로드를 진행합니다. 권한을 요청하면 허용합니다.

직접 명령으로 진행하려면 터미널에서 다음과 같이 입력합니다.

```bash
git add .
git commit -m "첫 커밋"
gh repo create space-invaders-내이름 --public --source=. --push
```

업로드 후 GitHub에서 저장소를 열면 `index.html` 과 `.github/` 폴더(지침·스킬 포함)가 보입니다.

## 2) 클라우드 에이전트에 작업 위임

### 방법 A — CLI에서 직접 위임

대화창에서 `/delegate` 로 작업을 클라우드에 맡깁니다.

```
/delegate 게임에 최고 점수 기록 기능을 추가해줘. localStorage로 최고 점수를 저장하고 화면(HUD)에 현재 점수와 함께 표시해줘. 최고 점수를 갱신하면 축하 메시지를 잠깐 보여줘. 모든 변경은 기존 단일 HTML 파일 안에서만 하고 새 파일이나 외부 의존성을 추가하지 마.
```

Copilot이 확인을 요청하면 위임을 승인합니다.

### 방법 B — GitHub 웹에서 이슈로 위임

1. GitHub 저장소 → **Issues** 탭 → **New issue**
2. 제목: `최고 점수 기록 기능 추가`
3. 본문: 방법 A의 요청문을 붙여넣기
4. 우측 **Assignees** 에서 **Copilot** 선택 → 이슈 생성

> 이슈 본문이 단일 파일 제약을 다시 명시하는 점에 주목하십시오. 이는 의도된 것으로, 에이전트에 명확한 수락 기준을 제공하는 동시에 지침 파일이 항상 적용되는 가드레일 역할을 합니다.

## 3) 결과(PR) 확인

- 잠시 후 클라우드 에이전트가 PR을 엽니다. GitHub의 **Pull requests** 탭 또는 **Agents** 탭에서 진행 과정을 실시간으로 확인할 수 있습니다.
- PR을 열어 다음을 검증합니다.
  - **단일 파일** — 기존 HTML 파일 안에서만 변경됐는가
  - **외부 의존성 없음** — CDN·패키지 추가 없이 구현됐는가
  - **기능 동작** — 최고 점수가 localStorage에 저장되고 HUD에 표시되는가

이 조건이 모두 충족되면, 작성한 지침이 클라우드 환경에서도 그대로 적용된 것입니다.

> 에이전트가 이슈 작업을 시작한 시점에 이미 목표는 달성된 것입니다. PR 생성에는 수 분이 걸릴 수 있습니다.

## 정리

프로젝트를 GitHub에 올리고 작업을 클라우드 에이전트에 위임했습니다. 에이전트는 기능을 자율적으로 구현하고 PR을 열되, 앞서 작성한 지침과 제약을 존중했습니다. 커스터마이징은 특정 도구나 편집기에 종속되지 않고 저장소를 따라 이동하며, Copilot이 동작하는 모든 환경에 동일하게 적용됩니다.
