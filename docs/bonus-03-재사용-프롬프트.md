# 보너스 3 — 재사용 프롬프트 (Prompt files)

지침이 항상 적용되는 규칙이고 스킬이 의도로 호출되는 워크플로우라면, 어떤 작업은 매번 동일하게 수동으로 호출하는 단순한 **온디맨드 프롬프트** 가 가장 적합합니다. **프롬프트 파일(Prompt files)** 이 그 용도입니다.

## 프롬프트 파일이란

프롬프트 파일은 `/명령` 으로 호출하는 재사용 가능한 마크다운 프롬프트입니다. 한 번 작성해 작업 공간(또는 개인 프로필)에 저장하면, 이후에는 여러 문단의 지시를 다시 입력하거나 기억할 필요 없이 `/이름` 으로 호출합니다.

프롬프트 파일은 `.prompt.md` 확장자와 약간의 YAML 설정 헤더를 가진 마크다운 파일입니다. 본문은 자연어로, 작업이 무엇인지, 사용자가 어떤 입력을 제공해야 하는지, 좋은 결과가 무엇인지, 어떤 제약을 지켜야 하는지를 기술합니다.

이점은 다음과 같습니다.

- **반복성** — 동일 작업을 매번 동일하게 기술해 결과가 일관됩니다.
- **공유성** — 저장소에 두면(작업 공간 범위) 모든 참가자가 별도 설정 없이 사용합니다.
- **품질** — 모범 사례, 스타일 가이드 링크, "완료 기준"을 프롬프트에 직접 담을 수 있습니다.

> 프롬프트 파일에도 범위가 있습니다. 작업 공간 프롬프트는 `.github/prompts/` 에 위치해 저장소 전체와 공유되고, 개인 프롬프트는 사용자 프로필에 위치해 모든 작업 공간에서 따라다닙니다.

## 작성 절차

`.prompt.md` 파일을 직접 손으로 쓰는 대신, **Copilot에게 프롬프트 파일을 만들어 달라고** 자연어로 요청합니다.

작업 폴더에서 새 대화를 시작한 뒤 다음을 입력합니다. (게임과 무관한, 실제 업무에 가까운 예시입니다.)

```
"회의록 정리기" 라는 작업 공간 프롬프트 파일(.prompt.md)을 만들어줘.
사용자가 회의록 텍스트 파일 하나를 첨부하면, 항상 같은 양식으로 정리해줘:
① 한 줄 요약 ② 결정 사항 ③ 할 일(담당자·기한 포함) ④ 다음 회의 안건.
회의록이 첨부되지 않으면 임의로 지어내지 말고 파일을 먼저 요청해줘.
```

> **참고:** Microsoft Build 원본 실습은 Visual Studio Code에서 `/create-prompt` 라는 내장 명령을 씁니다. 그건 **VS Code 전용** 명령이라 **Copilot CLI에는 없습니다.** CLI에서는 위처럼 평범한 자연어로 "이런 프롬프트 파일 만들어줘"라고 부탁하면 똑같은 `.prompt.md` 파일을 만들어 줍니다. (만들어진 `/명령` 호출은 CLI에서도 그대로 작동합니다.)

Copilot이 몇 가지를 되물을 수 있습니다(예: 입력이 없을 때의 동작, 양식 항목). 답한 뒤 제안된 파일을 검토하고 저장(Keep)합니다. 보통 `.github/prompts/회의록-정리기.prompt.md` 로 저장됩니다.

생성된 프롬프트 파일은 다음을 갖춰야 합니다.

- 작업 공간 아래 위치(팀과 공유)
- 최소한 설명(description)을 담은 YAML 설정 헤더
- 사용자가 회의록 파일 하나를 입력으로 제공해야 함을 명시하고, 입력이 없을 때의 동작(되묻기) 기술
- 항상 같은 출력 양식(요약·결정·할 일·다음 안건) 적용

> 핵심은 "항상 같은 양식으로"라는 **결과의 일관성**입니다. 한 번 만들어두면 누가 호출하든, 몇 번을 호출하든 동일한 형식의 회의록 요약이 나옵니다. 양식 기준을 사내 위키나 표준 문서 **URL**로 지정할 수도 있습니다 — Copilot이 그 페이지를 읽어 규칙을 반영합니다.

## 사용

1. 새 대화 시작:
   ```
   /clear
   ```
2. `/` 를 입력하면 방금 만든 **회의록-정리기** 가 명령 목록에 나타납니다. **이렇게 목록에 뜨는 것 자체가, 내가 만든 문서가 Copilot의 `/명령`으로 등록됐다는 증거입니다.**
3. 정리할 회의록 파일을 `#` 로 첨부해 실행합니다.
   ```
   /회의록-정리기 #회의록.txt
   ```
   `#회의록.txt` 는 해당 파일을 맥락으로 첨부하며, 이것이 프롬프트가 요구하는 입력입니다. 파일명이 다르면 그에 맞게 바꿉니다.
4. Copilot이 파일을 읽고, 프롬프트에 담긴 양식(요약·결정·할 일·다음 안건)대로 정리해 제안합니다. 검토 후 원하는 부분을 저장합니다.

> 파일을 `#` 로 첨부하지 않으면, 잘 작성된 프롬프트는 임의의 내용을 지어내는 대신 입력을 되물어야 합니다. 그렇지 않다면 프롬프트의 입력 명세를 더 분명히 해야 한다는 신호입니다.

## 정리

자연어 요청만으로 작업 공간 범위의 프롬프트 파일을 만들고, 입력에 대한 명확한 계약과 모범 사례 문서 링크를 담아, `/이름 #파일` 형태로 호출했습니다. 이제 저장소에서 작업하는 누구나 동일한 `/명령` 으로 일관된 결과를 얻습니다.

항상 적용되는 맥락인 **지침** 과, 필요할 때 호출하는 반복 작업인 **프롬프트 파일** — 잘 조정된 Copilot 설정의 두 축을 모두 갖췄습니다.
