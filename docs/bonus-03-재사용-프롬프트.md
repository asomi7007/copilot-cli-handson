# 보너스 3 — 재사용 프롬프트 (Prompt files)

지침이 항상 적용되는 규칙이고 스킬이 의도로 호출되는 워크플로우라면, 어떤 작업은 매번 동일하게 수동으로 호출하는 단순한 **온디맨드 프롬프트** 가 가장 적합합니다. **프롬프트 파일(Prompt files)** 이 그 용도입니다.

## 프롬프트 파일이란

프롬프트 파일은 `/명령` 으로 호출하는 재사용 가능한 마크다운 프롬프트입니다. 한 번 작성해 작업 공간(또는 개인 프로필)에 저장하면, 이후에는 여러 문단의 지시를 다시 입력하거나 기억할 필요 없이 `/이름` 으로 호출합니다.

프롬프트 파일은 `.prompt.md` 확장자와 약간의 YAML 설정 헤더를 가진 마크다운 파일입니다. 본문은 자연어로, 작업이 무엇인지, 사용자가 어떤 입력을 제공해야 하는지, 좋은 결과가 무엇인지, 어떤 제약을 지켜야 하는지를 기술합니다.

이점은 다음과 같습니다.

- **반복성** — 동일 작업을 매번 동일하게 기술해 결과가 일관됩니다.
- **공유성** — 저장소에 두면(작업 공간 범위) 모든 참가자가 별도 설정 없이 사용합니다.
- **품질** — 모범 사례, 스타일 가이드 링크, "완료 기준"을 프롬프트에 직접 담을 수 있습니다.

> 프롬프트 파일에도 범위가 있습니다. 작업 공간 프롬프트는 `.github/prompts/` 에 위치해 저장소 전체와 공유되고, 개인 프롬프트는 사용자 프로필에 위치해 모든 작업 공간에서 따라다닙니다.

## 작성 절차

`.prompt.md` 파일을 직접 쓰는 대신 `/create-prompt` 명령으로 생성합니다.

작업 폴더에서 새 대화를 시작한 뒤 다음을 입력합니다.

```
/create-prompt "꼼꼼한 문서 작성기" 라는 작업 공간 프롬프트를 만들어줘. 사용자가 지정한 파일 하나를 문서화하는데, 자바스크립트는 JSDoc 형식, 다른 언어는 해당 언어에 맞는 주석 형식을 사용해. 작성 규칙은 https://developer.mozilla.org/en-US/docs/MDN/Writing_guidelines/Code_style_guide/JavaScript 를 따라줘.
```

> `/create-prompt` 가 목록에 없으면 `awesome-copilot` 마켓플레이스에서 관련 도구를 설치하거나(대개 기본 제공됩니다) 진행자에게 문의하십시오.

Copilot이 몇 가지를 되물을 수 있습니다(예: 비자바스크립트 파일 처리, 입력 누락 시 동작). 답한 뒤 제안된 파일을 검토하고 저장(Keep)합니다. 보통 `.github/prompts/꼼꼼한-문서-작성기.prompt.md` 로 저장됩니다.

생성된 프롬프트 파일은 다음을 갖춰야 합니다.

- 작업 공간 아래 위치(팀과 공유)
- 최소한 설명(description)을 담은 YAML 설정 헤더
- 사용자가 단일 파일을 입력으로 제공해야 함을 명시하고, 입력이 없을 때의 동작 기술
- 규약의 기준으로 MDN 자바스크립트 스타일 가이드 참조
- 언어에 맞는 주석 형식 사용 지시(JS는 JSDoc, Python은 docstring, C#은 XML 주석 등)

> 요청에 외부 모범 사례 문서(MDN 스타일 가이드)의 URL을 포함한 점이 핵심입니다. Copilot이 그 페이지를 읽어 규칙을 반영합니다. 코딩 표준, 스타일 가이드, 내부 위키 등 어떤 URL이나 작업 공간 파일도 사용할 수 있습니다.

## 사용

1. 새 대화 시작:
   ```
   /clear
   ```
2. `/` 를 입력하면 방금 만든 **꼼꼼한-문서-작성기** 가 명령 목록에 나타납니다.
3. 문서화할 파일을 `#` 로 첨부해 실행합니다.
   ```
   /꼼꼼한-문서-작성기 #index.html
   ```
   `#index.html` 은 해당 파일을 맥락으로 첨부하며, 이것이 프롬프트가 요구하는 입력입니다. 파일명이 다르면 그에 맞게 바꿉니다.
4. Copilot이 파일을 읽고, 프롬프트에 담긴 규칙에 따라 함수에 주석을 추가하고 구조를 설명하는 변경을 제안합니다. 검토 후 원하는 부분을 저장합니다.

> 파일을 `#` 로 첨부하지 않으면, 잘 작성된 프롬프트는 임의의 파일을 고르는 대신 입력을 되물어야 합니다. 그렇지 않다면 프롬프트의 입력 명세를 더 분명히 해야 한다는 신호입니다.

## 정리

`/create-prompt` 로 작업 공간 범위의 프롬프트 파일을 만들고, 입력에 대한 명확한 계약과 모범 사례 문서 링크를 담아, `/이름 #파일` 형태로 호출했습니다. 이제 저장소에서 작업하는 누구나 동일한 `/명령` 으로 일관된 결과를 얻습니다.

항상 적용되는 맥락인 **지침** 과, 필요할 때 호출하는 반복 작업인 **프롬프트 파일** — 잘 조정된 Copilot 설정의 두 축을 모두 갖췄습니다.
