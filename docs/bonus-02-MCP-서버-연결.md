# 보너스 2 — MCP 서버 연결

본 실습에서 사용한 플러그인에는 커스텀 에이전트·스킬·훅과 함께 브라우저 자동화용 Playwright MCP 서버가 포함되어 있었습니다. 이 모듈에서는 **공용 갤러리(Community Hub)** 를 MCP 서버로 직접 연결하고, 그 서버가 노출하는 도구를 확인한 뒤 자연어로 사용합니다.

## MCP 서버란

**MCP(Model Context Protocol)** 는 Copilot이 **바깥 시스템에 손을 뻗어 할 수 있는 일(도구)을 늘려주는** 표준 연결 방식입니다. MCP 서버를 하나 연결하면, 그 서버가 제공하는 동작들이 Copilot의 새 능력으로 추가되고, 필요할 때 Copilot이 알아서 사용합니다.

MCP는 Copilot이 지금 작업 폴더 **밖**에 있는 것에 닿아야 할 때 쓰입니다.

- **바깥 데이터 읽기** — 데이터베이스, 이슈 트래커, 대시보드, 사내 시스템에서 정보를 가져온다.
- **바깥 동작 실행** — 파일 업로드, 기록 생성 같은 작업을 대신 수행한다.
- **외울 필요 없음** — 어떤 주소로 어떻게 요청하는지 사람이 외울 필요 없이, Copilot이 서버에 무슨 기능이 있는지 스스로 확인한다.

이 실습의 공용 갤러리는 다음 주소로 MCP 서버를 공개합니다.

```
https://rg-hub-web-0614.azurewebsites.net/mcp
```

## 1) MCP 서버 추가

대화창에서 MCP 추가 명령을 사용합니다(버전에 따라 `/mcp add` 또는 `/mcp`).

```
/mcp add
```

안내가 나오면 다음과 같이 입력합니다.

- 유형: **HTTP**
- 주소: `https://rg-hub-web-0614.azurewebsites.net/mcp`
- 이름: `lab502-community-hub`

> 자연어로 요청할 수도 있습니다.
> ```
> https://rg-hub-web-0614.azurewebsites.net/mcp 를 lab502-community-hub 라는 이름의 HTTP MCP 서버로 추가해줘.
> ```

추가 후 `/env` 또는 `/mcp` 로 연결 상태와 도구 개수를 확인합니다.

## 2) 노출된 도구 확인

대화창에서 다음을 요청합니다.

```
lab502-community-hub MCP 서버가 제공하는 도구들이 무엇이고 각각 어떤 일을 하는지 정리해줘.
```

다음과 같은 도구들이 표시됩니다.

- 공유된 게임 목록 조회
- 스크린샷 목록 조회
- 커뮤니티 활동 조회
- 도구 사용량 조회
- 게임·스크린샷 업로드 방법 안내

## 3) 자연어로 사용

도구 이름을 외울 필요 없이, 원하는 바를 말하면 Copilot이 적절한 도구를 선택합니다.

```
공용 갤러리에 최근 공유된 우주 침략자 게임 목록과 주소를 보여줘.
```
```
지금까지 갤러리에 올라온 스크린샷들을 보여줘.
```
```
최근 공용 갤러리 활동을 확인해 참가자들의 작업을 요약해줘.
```

> 도구 호출을 펼쳐 보면 실제 입력과 출력을 확인할 수 있어, 자신의 프롬프트에서 이 도구를 활용하는 방법을 이해하는 데 도움이 됩니다.

## 플러그인 방식과 MCP 방식 비교

외부 능력을 Copilot에 연결하는 두 가지 방식을 보았습니다.

- **플러그인 방식**(본 실습) — 에이전트·스킬이 세부 사항을 감싸므로 "화면을 캡처해 공유해줘" 한 문장으로 끝납니다. 정해진, 의견이 반영된 워크플로우에 적합합니다.
- **MCP 직접 연결 방식**(이 보너스) — 도구를 직접 노출하므로, 데이터를 살펴보거나 즉석에서 질문하거나 자신만의 워크플로우를 구성할 때 적합합니다.

둘은 대체재가 아니라 상호 보완적입니다. MCP는 호스트와 서버가 인증·권한을 표준화된 방식으로 처리하게 해 주는데, 이는 스킬마다 자격 증명 흐름을 따로 구현하기 어려운 부분을 해결합니다. 강력한 커스터마이징은 보통 둘을 결합합니다. 에이전트·스킬이 워크플로우를 제공하고, MCP 서버가 도구를 공급합니다.
