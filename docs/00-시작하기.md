# 00 — 시작하기

이 실습에서는 빈 폴더에서 출발해 "우주 침략자(Space Invaders)" 게임을 생성하고, 다른 참가자들과 공유한 뒤, GitHub Copilot을 자신의 작업 방식에 맞게 조정하는 방법을 다룹니다. 모든 작업은 터미널에서 자연어로 진행하며, 코드 작성 지식은 필요하지 않습니다.

## 학습 목표

이 실습을 마치면 다음을 할 수 있습니다.

- 터미널(CLI)에서 자연어로 Copilot에 작업을 지시한다.
- **계획(Plan) 모드**로 복잡한 작업을 코드 작성 전에 설계한다.
- Copilot의 동작을 조정하는 구성 요소를 이해하고, 그중 **지침(Instructions)**·**스킬(Skills)** 등을 직접 만든다.

이 구성 요소를 문제에 맞게 선택하는 기준은 다음과 같습니다.

- **지침(Instructions)** — 프로젝트 전반에 항상 적용되는 규칙
- **프롬프트 파일(Prompt files)** — 반복 실행하는 온디맨드 명령
- **스킬(Skills)** — 의도에 따라 자동 실행되거나 명령으로 호출하는 여러 단계 작업
- **커스텀 에이전트·훅·MCP 서버** — 그 외의 동작을 담당하는 요소

## 실습 흐름

1. **환경 준비** — Copilot CLI를 실행하고 GitHub에 로그인한다.
2. **플러그인 설치** — 커뮤니티 마켓플레이스에서 게임 공유 도구를 설치한다.
3. **게임 생성** — 계획 모드로 단일 HTML 파일 게임을 생성한다.
4. **게임 공유** — 커스텀 에이전트로 화면을 캡처해 공용 갤러리에 공유한다.
5. **구성 요소 확인** — Copilot을 조정하는 요소들을 살펴본다.
6. **지침·스킬 작성** — 저장소 지침과 에이전트 스킬을 직접 만든다.

## 보너스 흐름

시간이 남으면 같은 맥락을 이어가는 보너스 모듈을 진행합니다.

1. **클라우드 에이전트** — 저장소 지침을 GitHub에서 적용한다.
2. **MCP 서버 연결** — 공용 갤러리를 MCP 서버로 연결해 그 도구를 직접 사용한다.
3. **재사용 프롬프트** — 온디맨드 명령으로 프롬프트 파일을 만든다.

> Copilot은 대규모 언어모델(LLM)을 사용하므로 응답을 확률적으로 생성합니다. 따라서 제안·코드·동작이 이 문서와 다소 다를 수 있으며, 이는 정상입니다. 첫 응답이 적절하지 않으면 다시 요청해 조정하면 됩니다.
