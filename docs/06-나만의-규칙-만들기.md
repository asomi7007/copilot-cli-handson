# 06 — 지침(Instructions) 작성

이 모듈에서는 Copilot이 항상 따르는 규칙인 **지침(Instructions)** 을 작성합니다. 한 번 만들어 두면 이 폴더에서 작업할 때마다 Copilot이 해당 규칙을 자동으로 적용합니다.

## 지침이란

지침은 프로젝트의 "기준 규칙"을 담은 마크다운 문서로, Copilot이 이 작업 공간에서 무엇이 올바른지 이해하는 데 사용하는 지속적 안내입니다. 저장소 수준에는 두 종류가 있습니다.

- **copilot-instructions.md** — 메인 지침 파일. 해당 작업 공간의 모든 요청에 Copilot이 **항상** 불러옵니다.
- **.instructions.md** — 경로별 지침. 설정 헤더의 `applyTo` 글롭으로 범위를 지정합니다. 항상 불러오지 않고, 요청이나 관련 파일이 범위와 일치할 때만 적용됩니다.

일회성 요청과 달리 `copilot-instructions.md` 는 항상 적용되는 맥락입니다. 한 번 작성하면 이후 모든 세션이 동일한 기준 안내를 받습니다. 이것이 "Copilot이 적당히 추측한" 결과와 "이 프로젝트에 일관되게 들어맞는" 결과의 차이를 만듭니다.

좋은 지침은 보통 코드만으로는 알 수 없는 내용을 담습니다.

- 프로젝트의 목적과 상위 구조
- 규칙과 제약 — 코딩 스타일, 명명 규칙, 선호/지양 라이브러리, 파일 구성
- 실행·테스트 방법
- 피해야 할 사항(안티패턴, 폐기된 API, 건드리면 안 되는 파일)

> 이 게임은 자체 완결된 단일 HTML 파일입니다. 이는 중요한 제약입니다. 명시하지 않으면 Copilot이 다음 변경에서 파일을 여러 개로 분리하거나 빌드 단계를 추가하거나 외부 의존성을 끌어올 수 있습니다. 이 제약을 지침에 담아 이후 요청이 자동으로 따르도록 합니다.

## 지침의 범위

지침은 세 가지 범위로 나뉘며 조합할 수 있습니다.

- **개인** — 모든 작업 공간에서 사용자 본인에게 적용
- **저장소** — 이 저장소에서 작업하는 모두에게 적용, `.github/copilot-instructions.md` 에 저장
- **조직** — 조직 내 모든 저장소에 적용, 관리자가 설정

> 지침은 특정 편집기에 종속된 기능이 아닙니다. 저장소 지침은 코드와 함께 이동하며 Copilot CLI, 편집기, GitHub 웹, 클라우드 에이전트 등 모든 환경에서 동일하게 적용됩니다.

## 작성 절차

마크다운 파일을 직접 작성할 수도 있지만, 빈 파일에서 시작하는 것보다 `/create-instructions` 명령으로 초안을 생성한 뒤 다듬는 편이 빠릅니다.

작업 폴더(`invaders`) 안인지 확인하고, 생성된 HTML 파일이 폴더에 있는 상태에서(분석 대상이 있어야 결과가 좋습니다) 다음을 입력합니다.

```
/create-instructions .github/copilot-instructions.md 파일을 만들어줘. 이 게임은 항상 외부 의존성 없는 단일 HTML 파일 하나로 유지되어야 해.
```

> `/create-instructions` 가 목록에 없으면 `awesome-copilot` 마켓플레이스에서 관련 플러그인을 설치하거나(대개 기본 제공됩니다) 진행자에게 문의하십시오.

- Copilot이 작업 공간을 분석하고, 범위·규칙에 관해 몇 가지를 되묻거나 도구 실행 권한을 요청할 수 있습니다. 답하거나 허용합니다.
- 제안된 파일을 검토한 뒤 저장(Keep)합니다.

> 핵심 제약만 직접 제시했고, 나머지 지침은 Copilot이 코드와 대화 이력에서 추론합니다. 요청이 구체적일수록 생성 결과가 좋아집니다.

## 검토 및 보강

첫 초안은 출발점입니다. `.github/copilot-instructions.md` 를 열어(또는 "그 파일 내용을 보여줘"로 확인) 다음이 포함됐는지 점검합니다.

- 프로젝트 한 단락 설명(게임, 단일 HTML 파일, 테마)
- **단일 파일 제약** — HTML·CSS·JS가 한 파일에 인라인, 빌드 단계 없음, 외부 자산·CDN 미사용
- 실행 방법(브라우저로 HTML 파일 열기)
- 스타일·시각 규칙(테마, 색상, 외부 폰트 미사용 등)
- Copilot이 하지 말아야 할 것(모듈 분리, 패키지 추가, 프레임워크 도입)

빠진 부분이 있으면 파일을 직접 수정하거나 Copilot에 보강을 요청합니다.

```
.github/copilot-instructions.md 에 외부 인터넷 자료(CDN)를 사용하지 말라는 규칙을 추가하고, 빌드 단계와 외부 의존성도 금지한다고 명시해줘.
```

## 적용 확인

1. 새 대화를 시작해 이전 맥락을 비웁니다.
   ```
   /clear
   ```
2. 작은 변경을 요청합니다.
   ```
   게임 화면 하단에 "Microsoft Build 2026에서 제작" 이라는 작은 문구를 추가해줘.
   ```
3. Copilot이 기존 단일 HTML 파일 안에서 수정하면 지침이 적용된 것입니다. 새 파일을 만들거나 외부 자료를 끌어오면, 지침을 더 구체적으로 다듬거나 게임 HTML 파일을 맥락에 명시적으로 추가해야 한다는 신호입니다.

> 지침은 강제 제약이 아니라 안내입니다. 대부분의 경우 따르지만, 모호한 요청에서는 벗어날 수 있습니다. 해결책은 거의 항상 해당 규칙을 더 구체적으로 작성하는 것입니다.
