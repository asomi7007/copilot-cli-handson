# 05 — 구성 요소 확인

지금까지는 다른 사람이 만든 플러그인을 사용했습니다. 그 플러그인 안에는 Copilot의 동작을 조정하는 여러 구성 요소가 들어 있습니다. 이 모듈에서는 그 요소들의 정식 명칭과 역할을 정리합니다. 직접 만들거나 외부에 연결하는 작업은 06·07 모듈과 보너스 모듈에서 다룹니다.

> 이 실습의 목표 중 하나가 바로 이 부분입니다. 각 구성 요소가 무엇이고, 어떻게 만들고 수정해 Copilot을 자신의 작업 방식에 맞추는지 이해하는 것입니다.

## Copilot을 구성하는 9가지 요소

정식 명칭(영문)과 역할을 함께 정리한 표입니다.

| # | 정식 명칭 | 역할 | 작동 시점 | 이 실습에서 |
|---|----------|------|----------|------------|
| 1 | **지침 (Instructions)** | 프로젝트에 항상 적용되는 규칙 | 매 요청마다 자동 | 06에서 직접 생성 |
| 2 | **프롬프트 파일 (Prompt files)** | `/명령` 으로 호출하는 반복 워크플로우 | 명령으로 호출 시 | 보너스 3에서 생성 |
| 3 | **커스텀 에이전트 (Custom agents)** | 전용 도구·성격을 가진 특화 보조자 | 선택 또는 위임 시 | 04에서 사용 |
| 4 | **스킬 (Skills)** | 여러 단계로 된 작업 묶음 | 의도 일치 시 자동 | 07에서 직접 생성 |
| 5 | **도구 (Tools)** | 검색·읽기·편집·명령 실행 등 실제 능력 | 작업에 필요할 때 | 게임 생성 시 내부 사용 |
| 6 | **서브에이전트 (Subagents)** | 작업 일부를 별도 에이전트에 위임 | Copilot 판단 시 | (자동) |
| 7 | **훅 (Hooks)** | 정해진 시점에 자동 실행되는 동작 | 라이프사이클 이벤트 시 | 플러그인이 활동 기록에 사용 |
| 8 | **MCP 서버** | 외부 시스템·API·데이터에 연결 | 외부 정보·동작 필요 시 | 보너스 2에서 연결 |
| 9 | **플러그인 / 워크플로우** | 위 요소들의 묶음 / 저장소 자동화 | 설치 또는 이벤트 시 | 02에서 설치 |

이 요소들은 경쟁 관계가 아니라 상호 보완 관계입니다. 지침이 기반 규칙을 깔고, 프롬프트·에이전트가 워크플로우를 패키징하고, 스킬이 작업 능력을 더하고, 도구가 실제로 실행하고, 서브에이전트가 복잡한 작업을 분리하고, 훅이 자동화를 더하고, MCP가 외부로 확장하고, 플러그인이 이 전체를 묶어 공유합니다.

## 선택 기준

- 프로젝트에 대해 Copilot이 항상 알아야 하는 내용 → **지침**
- `/명령` 으로 동일 작업을 반복 → **프롬프트 파일**(단순) 또는 **스킬**(여러 단계)
- 에이전트가 적용 시점을 스스로 판단 → **스킬**(설명을 잘 작성하면 자동 인식)
- 특정 작업용 전용 보조자 → **커스텀 에이전트**
- 모델이 잊더라도 매번 확실히 실행 → **훅**
- 외부 서비스·데이터 연결 → **MCP 서버**
- 위 요소들을 묶어 공유·배포 → **플러그인**

## 현재 적용된 구성 확인

대화창에서 다음을 입력하면 현재 세션에 적용된 지침·스킬·에이전트·플러그인·MCP가 표시됩니다.

```
/env
```

02 모듈에서 설치한 **space-invaders-makers** 와 그 안의 에이전트·스킬이 보입니다.

## 스킬의 내부 구조 확인

스킬은 사람이 읽을 수 있는 문서입니다. `SKILL.md` 파일에 "언제 이 스킬을 쓰고 어떤 순서로 동작하는지"가 자연어로 기술되어 있습니다. 직접 확인하려면 대화창에서 다음과 같이 요청합니다.

```
설치된 share-screenshot 스킬의 SKILL.md 내용을 보여줘
```

표시되는 내용에서 다음을 확인할 수 있습니다.

- 상단의 **이름(name)** 과 **설명(description)** — Copilot이 이 스킬을 언제 사용할지 판단하는 기준입니다.
- 그 아래의 **자연어 작업 지시** — 특수한 형식이 아니라 단계별 산문입니다.

> 파일을 편집기로 열어 확인할 수도 있습니다("그 파일을 메모장으로 열어줘"). 내용을 보는 것이지 수정하는 것이 아니므로, CLI로 보든 편집기로 보든 동일합니다.
