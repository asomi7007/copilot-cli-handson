# 04 — 게임 공유

생성한 게임의 화면을 직접 캡처해 업로드하는 대신, 작업 전체를 플러그인에 포함된 **web-screenshotter 커스텀 에이전트**에 위임합니다.

## 커스텀 에이전트

커스텀 에이전트는 YAML 설정 헤더가 붙은 마크다운 문서로 정의되는 특화된 보조자입니다(03 모듈에서 본 `web-screenshotter.agent.md`). 각 에이전트는 다음을 가집니다.

- 자연어로 기술된 **명확한 목적**
- 사용 가능한 **도구의 명시적 목록**(그리고 그 도구만 사용)
- 선택적으로, 실행 시 자동으로 기동되는 **자체 MCP 서버**
- 선택적으로, 상위 동작을 구성하기 위한 **스킬** 접근

메인 세션에 직접 시키지 않고 별도 에이전트를 쓰는 이유는 다음과 같습니다.

- **격리** — 보조 에이전트는 깨끗한 맥락에서 필요한 도구만 가지고 동작합니다. 메인 세션이 브라우저 자동화 호출이나 캡처 데이터로 복잡해지지 않습니다.
- **재사용성** — 같은 에이전트를 우리 게임뿐 아니라 임의의 URL이나 HTML 파일에도 쓸 수 있습니다.
- **예측 가능성** — 에이전트 지시에 엄격한 규칙(예: "공유에 성공해야 작업 완료")이 들어 있어 워크플로우가 일관됩니다.
- **구성** — 에이전트가 MCP 서버(Playwright)와 스킬(share-screenshot)을 조합해 동작합니다. 사용자는 각각의 동작 방식을 몰라도 됩니다.

요약하면, 사용자는 원하는 결과("이 화면을 캡처해 공유")만 말하고, 방법은 에이전트가 처리합니다.

## 공유 절차

게임을 생성한 동일한 대화 세션에서 진행합니다.

1. 자동 실행을 허용해, 에이전트 동작 중 매번 확인을 받지 않도록 합니다.
   ```
   /allow-all on
   ```

2. 에이전트 선택 창을 엽니다.
   ```
   /agent
   ```
   목록에 **Default (current)** 와 **web-screenshotter** 가 표시됩니다. **web-screenshotter** 를 번호 또는 방향키로 선택합니다. 이 에이전트는 02 모듈에서 플러그인을 설치할 때 사용 가능해졌습니다.

   > 진행자가 **Codespaces** 환경을 안내했다면, 화면이 없는 환경용인 **cs-web-screenshotter** 를 선택합니다.

3. 자연어로 요청합니다.
   ```
   생성한 우주 침략자 게임 화면을 캡처해서 공유해줘. 캡처 전에 게임을 시작해줘.
   ```

   이 요청에는 Playwright, URL, 파일 경로, 업로드 주소가 전혀 언급되지 않은 점에 주목하십시오. 게임을 생성한 동일 세션이므로 Copilot이 이미 파일 맥락을 가지고 있고, 요청이 web-screenshotter 에이전트의 설명과 일치하므로 작업을 위임합니다.

4. 에이전트의 동작을 확인합니다. Playwright MCP 서버가 비헤드리스 모드로 설정되어 있어, 실제 브라우저 창이 열립니다. 에이전트는 다음을 수행합니다.
   - 작업 폴더에서 생성된 HTML 파일 경로를 찾는다.
   - 브라우저로 연다.
   - 게임을 시작한다(키 입력 또는 버튼 클릭).
   - 전체 화면을 임시 파일로 캡처한다.
   - share-screenshot 스킬을 호출해 공용 갤러리에 업로드한다.

   진행 중 브라우저 창을 닫지 마십시오. 닫으면 에이전트가 실패합니다.

5. 완료되면 에이전트가 성공과 함께 업로드 결과를 보고합니다.

## 확인하기

올라간 **스크린샷**은 **활동 보드**에서 바로 볼 수 있습니다. 브라우저에서 엽니다(`<주소>` = 진행자가 안내한 갤러리 주소).

- **활동 보드:** `<주소>/activity`

내 스크린샷뿐 아니라 다른 참가자가 올린 것도 함께 실시간으로 표시됩니다.

> **맥락 참고:** Copilot에 게임 HTML 파일의 위치를 알려주지 않았습니다. 동일 세션의 대화 이력에 파일 경로가 이미 포함되어 있기 때문입니다. 새 세션에서 이 작업을 하거나 `/clear` 이후라면 그 맥락이 사라지므로, 경로를 직접 알려주거나(`@경로/index.html 캡처해서 공유해줘`) Copilot이 작업 폴더를 탐색하도록 해야 합니다.

> **선택:** `/chronicle` 을 입력하면 사용 가능한 옵션이 표시됩니다. `/chronicle standup` 은 수행한 작업을 요약합니다. 대화를 내보내려면 `/share file` 또는 `/share gist` 를 사용합니다.

## 내부적으로 일어난 일

짧고 모호한 한 문장이 플러그인에 선언된 여러 구성 요소를 연쇄적으로 작동시켰습니다.

- **메인 세션** 이 요청을 web-screenshotter 에이전트의 설명과 대조해 작업을 위임했습니다.
- **web-screenshotter 에이전트** 가 실행 정책(이동 → 상호작용 → 캡처 → 공유 → 검증)을 따랐습니다.
- **Playwright MCP 서버** 가 브라우저 자동화 도구를 제공했습니다(에이전트가 `npx` 로 자동 기동).
- **share-screenshot 스킬** 이 업로드를 수행했습니다(플랫폼에 맞는 스크립트 실행, 종료 코드로 성공/실패 판정).
- **플러그인 훅** 이 그 동안 활동 이벤트(세션 시작, 도구 사용)를 공용 갤러리로 조용히 전송했습니다.

에이전트·MCP 서버·스킬·훅을 하나의 플러그인에 구성하면, 한 줄의 자연어 요청이 신뢰할 수 있는 여러 단계 자동화가 됩니다. 사용자는 그 아래의 도구를 전혀 몰라도 됩니다.
