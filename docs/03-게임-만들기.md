# 03 — 게임 만들기

자연어 요청으로 게임을 생성합니다. 이 모듈에서는 두 가지를 다룹니다. 작업을 코드 작성 전에 설계하는 **계획(Plan) 모드**, 그리고 앞서 설치한 **플러그인이 내부적으로 동작하는 방식**입니다.

> Copilot은 대규모 언어모델(LLM)이라 응답을 확률적으로 생성합니다. 따라서 매번 동일한 코드나 결과가 나오지 않습니다. 화면이 이 문서와 다소 달라도 정상이며, 결과가 마음에 들지 않으면 다시 요청해 수정하면 됩니다.

---

## 계획(Plan) 모드

GitHub Copilot CLI에는 두 가지 작동 모드가 있습니다. 둘 다 도구 호출 기능을 모두 사용할 수 있습니다.

- **에이전트 모드(기본)** — 요청을 받으면 곧바로 구현을 시작합니다. 실행 모드라고도 합니다.
- **계획(Plan) 모드** — 코드를 작성하기 전에 먼저 설계를 협의합니다.

계획 모드에서는 진행이 대화형입니다. Copilot이 후속 질문을 던져(예: 기능 범위, 난이도) 의도를 확인하고 설계 결정을 함께 정합니다. 방향이 합의되면 구조화된 실행 계획을 제시하고, 사용자가 승인한 뒤에야 코드 작성을 시작합니다.

계획 모드의 이점은 다음과 같습니다.

- 오해를 초기에 발견할 수 있습니다(완성 후 재작업 방지).
- 구현 방식을 근거를 가지고 결정할 수 있습니다.
- 여러 단계로 이루어진 복잡한 작업에서 통제권을 유지할 수 있습니다.

모드 전환은 `Shift+Tab` 으로 일반 → 계획 → 자동(Autopilot) 순으로 순환하거나, 문장 앞에 `/plan` 을 붙이면 계획 모드로 바로 전환됩니다.

## 게임 생성

### 1) Copilot 실행

이전 세션을 종료했다면 작업 폴더(`copilot-workspace\invaders`, 01 모듈 참고)에서 다시 실행합니다.

```bash
copilot
```

### 2) 모델 확인 (선택)

화면 우측 하단 상태 표시줄에 현재 사용 중인 모델이 표시됩니다. 빠른 결과를 위해 **낮은 노력(low effort)** 설정을 권장합니다. 모델을 변경하려면 `/model` 을 입력해 선택합니다. 권장 모델은 **GPT-5.4** 또는 **Claude Sonnet 4.6**, 노력 수준은 **low** 입니다. 기본값을 그대로 두어도 무방합니다.

### 3) 계획 모드로 게임 요청

문장 앞에 `/plan` 을 붙입니다. **단일 HTML 파일로 생성**하도록 요청하는 것이 중요합니다. 파일이 하나면 어디서든 바로 열 수 있어 공유가 간편하기 때문입니다.

```
/plan 우주 침략자(Space Invaders) 게임을 하나의 HTML 파일로 만들어줘
```

계획을 생성할 때는 구체적으로 요청할수록 정확한 결과가 나옵니다. 테마나 기능을 덧붙일 수 있습니다.

- `... 바닷속 테마로, 물고기가 적이고 잠수함이 주인공이야`
- `... 하이스코어 기록, 여러 레벨, 효과음, 모바일 지원을 포함해줘`
- `... 한국 전통 단청 색감으로`

> **웹 자료로 맥락 제공:** Copilot은 웹 콘텐츠를 읽어와 참고할 수 있습니다.
> ```
> /plan 우주 침략자 게임을 하나의 HTML 파일로 만들어줘. https://www.microsoft.com/ko-kr/build 의 색상과 분위기를 참고해 Microsoft Build 테마로 구성해줘.
> ```

요청이 모호하면 Copilot은 두 가지 중 하나로 반응합니다. (1) 요구사항을 명확히 하기 위해 되묻거나, (2) 가정을 세워 계획을 생성합니다. 둘 다 정상이며, 가정이 틀렸거나 질문이 적절치 않으면 피드백으로 바로잡으면 됩니다.

### 4) 계획 검토 및 승인

계획이 완성되면 "**Plan Ready for Review**"(계획 검토 준비됨)가 표시됩니다. 이때 선택지는 다음과 같습니다.

- **Accept plan and build on autopilot**(계획 승인 후 자동 빌드) — 권장 선택입니다. 중간 확인 없이 끝까지 구현합니다.
  - 이 선택지가 없고 **Accept plan and build on default permissions** 만 보이면, 그것을 선택한 뒤 `/allow-all on` 을 입력해 자동 모드로 전환합니다.

나머지 선택지는 다음과 같습니다(이번에는 사용하지 않아도 됩니다).

- **Accept plan and build on default permissions** — 구현하되 권한이 필요한 단계마다 확인을 요청합니다. 실행을 더 통제하고 싶을 때 적합합니다.
- **Suggest changes** — 계획이 의도와 다르면 수정을 요청합니다(예: 효과음 추가, 테마 변경).
- **Exit plan mode** — 계획을 유지한 채 사용자가 직접 진행합니다.

## 생성된 게임 실행

Copilot이 계획 실행을 마치면 일반적으로 "**Task complete**"(작업 완료)가 표시되고, 작업 폴더에 HTML 파일 하나(예: `index.html`)가 생성됩니다.

브라우저로 열려면 대화창에 다음과 같이 입력합니다. `!` 는 셸 명령을 실행하라는 의미이고, `@` 는 파일 이름 자동완성을 띄웁니다.

```
! @
```

`@` 입력 후 자동완성 목록에서 생성된 HTML 파일을 선택하거나 파일 이름을 직접 입력합니다. 열 프로그램을 묻는 창이 나오면 웹 브라우저를 선택합니다.

한 번 실행해 동작을 확인하고, 수정이 필요하면 자연어로 요청합니다.

- `적의 이동 속도를 줄여줘`
- `배경을 더 어둡게 바꿔줘`
- `발사 효과음을 추가해줘`

> **시간이 부족한 경우:** 진행자가 미리 생성해 둔 게임 파일을 작업 폴더에 복사해 열어도 됩니다. 다만 계획 모드를 직접 체험하는 것이 이 모듈의 핵심이므로 한 번은 직접 생성해 보길 권합니다. 미리 만든 파일을 쓰면 이후 모듈의 일부 안내에서 파일 이름만 본인 것으로 바꾸면 됩니다.

## 참고: 설치한 플러그인의 내부 구조

게임이 생성되는 동안, 02 모듈에서 설치한 **space-invaders-makers** 플러그인이 무엇으로 구성되는지 살펴봅니다. 게임 생성과는 무관하므로 건너뛰어도 되지만, 이후 모듈에서 보게 될 동작의 배경을 이해하는 데 도움이 됩니다. 플러그인은 네 부분으로 구성됩니다.

### 에이전트(agents) — web-screenshotter

게임 화면을 캡처해 공유하는 전용 에이전트입니다. 브라우저 자동화 도구(Playwright)로 페이지에 접속해 전체 화면을 캡처하고, 공유 기능을 자동으로 호출합니다(04 모듈에서 사용).

에이전트는 설정 헤더(front matter)에 자신의 정체와 사용 가능한 도구를 선언합니다. 핵심은 **허용된 도구만 사용**하도록 제한한다는 점입니다. 이 에이전트에는 브라우저 도구와 실행 도구만 허용되어, 불필요한 동작을 막는 안전 경계 역할을 합니다.

본문에는 엄격한 실행 규칙이 자연어로 기술되어 있습니다.

- 공유는 필수다. 캡처만 하고 공유에 실패하면 작업이 완료된 것이 아니다.
- 실제로 공유에 성공한 경우에만 "공유했다"고 보고한다.
- 실패 시 한 번 재시도하고, 그래도 실패하면 중단하고 실패 사유를 보고한다.

> 에이전트는 이렇게 정의합니다. 설정 헤더(정체와 사용 도구) + 자연어로 기술한 동작 규칙. 별도의 프로그래밍 언어가 아니라, Copilot이 읽고 실행하는 구조화된 문서입니다. 도구 목록 선언은 선택이지만, 명시하면 에이전트가 필요한 도구만 쓰도록 제한해 동작을 예측 가능하게 만듭니다.

### 스킬(skills) — share-screenshot

캡처한 이미지를 공용 갤러리에 업로드하는 기능입니다. 위 에이전트가 캡처 후 호출하지만, 사용자가 직접 호출해 임의의 이미지를 공유할 수도 있습니다.

스킬은 `SKILL.md` 라는 문서로 정의됩니다. 상단에 이름·설명·인자 힌트(전달할 값, 여기서는 이미지 파일 경로)가 있고, 본문에 단계별 작업 지시가 자연어로 기술됩니다. 실제 업로드 같은 정확성이 필요한 작업은 별도의 스크립트가 처리하고, 스킬 문서는 "무엇을, 언제" 하는지를 정의합니다.

> 스크립트로 분리하는 이유는 신뢰성입니다. HTTP 요청·재시도 같은 정확한 작업을 매 실행마다 모델이 즉석에서 작성하면 불안정합니다. 스크립트에 위임하면 동작이 일관되고 빠르며 토큰도 절약됩니다. 모델은 "언제 실행할지" 판단하고, 스크립트는 "어떻게"를 담당합니다.

### 스크립트(scripts)

훅이 실행하는 작은 프로그램들입니다. 세션 시작·프롬프트 제출·도구 사용 같은 이벤트 정보를 공용 갤러리 서버로 전송합니다. 이 데이터가 실시간 활동 보드(현재 접속자 수, 자주 호출되는 도구 등)를 채웁니다.

### 훅(hooks)

특정 시점에 자동으로 실행되는 동작을 정의합니다. 이 플러그인은 네 가지 시점에 반응합니다.

- **세션 시작(SessionStart)** — Copilot 세션이 시작될 때
- **프롬프트 제출(UserPromptSubmit)** — 사용자가 요청을 보낼 때
- **도구 사용 후(PostToolUse)** — 에이전트가 도구를 사용한 직후
- **서브에이전트 종료(SubagentStop)** — 보조 에이전트가 종료될 때

각 시점에 위 스크립트를 실행해 활동을 서버에 보고합니다. 훅은 활동 기록 외에도 도구 사용을 차단하거나 추가 맥락을 주입하는 데 쓸 수 있지만, 여기서는 기록 용도로만 사용합니다.

> 정리하면 에이전트·스킬·스크립트·훅이 하나의 플러그인 안에서 함께 동작합니다. 그래서 04 모듈에서 "게임 화면을 캡처해 공유해줘"라는 한 문장이 이 요소들을 연쇄적으로 작동시켜 작업을 완료합니다.
