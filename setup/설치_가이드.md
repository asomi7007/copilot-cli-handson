# 🛠️ 설치 가이드 (참가자용)

GitHub Copilot CLI를 내 컴퓨터에 설치합니다. **한 번만** 하면 됩니다.

> 필요한 것: ① 인터넷 ② **GitHub 계정** ③ **GitHub Copilot 구독**(무료 체험 포함). 구독이 없으면 로그인 단계에서 막힙니다. 미리 https://github.com/features/copilot 에서 확인하세요.

---

> 📌 설치 스크립트(`install.ps1` / `install.sh`)는 **영문**입니다. 한글 파일명·콘솔 메시지가 일부 환경(PowerShell 등)에서 깨지는 문제를 피하기 위함입니다. 이 가이드 문서는 한글입니다.

## 방법 1: 한 줄 명령으로 바로 설치 (권장)

GitHub에서 설치 스크립트를 받아 바로 실행합니다.

### 🪟 Windows (PowerShell)

1. 시작 메뉴에서 **PowerShell** 검색 → **마우스 오른쪽 → "관리자 권한으로 실행"**
   > ⚠️ **반드시 관리자 권한으로** 여세요. Node 설치와 전역(npm -g) 설치에 권한이 필요합니다. 일반 권한이면 스크립트가 안내 후 멈춥니다.
2. 아래 한 줄을 붙여넣고 Enter (실행 정책 우회 + 다운로드 실행이 한 줄에 포함):
   ```powershell
   Set-ExecutionPolicy Bypass -Scope Process -Force; Invoke-RestMethod https://raw.githubusercontent.com/asomi7007/copilot-cli-handson/main/setup/install.ps1 | Invoke-Expression
   ```
   > 참고: `Invoke-RestMethod`(별칭 `irm`)는 PowerShell에 기본 내장돼 있습니다. `wget`은 윈도우 PowerShell에서 다르게 동작하므로 쓰지 않습니다.

### 🍎 Mac (터미널)

1. **터미널** 앱 열기 (스팟라이트에서 `터미널` 검색)
2. 아래 한 줄을 붙여넣고 Enter (sudo 불필요):
   ```bash
   curl -fsSL https://raw.githubusercontent.com/asomi7007/copilot-cli-handson/main/setup/install.sh | bash
   ```
   > Mac은 Homebrew가 사용자 영역에 설치하므로 관리자(sudo)가 필요 없습니다. 전역 npm 설치에서 권한 오류가 나면 스크립트가 `sudo` 대안을 안내합니다. (스크립트 전체를 sudo로 실행하지 마세요 — Homebrew가 거부합니다.)

설치가 끝나면 화면에 다음 단계 안내(영문)가 나옵니다. 그대로 따라가면 됩니다.

## 방법 1-B: 저장소를 받아서 실행

```bash
git clone https://github.com/asomi7007/copilot-cli-handson.git
cd copilot-cli-handson/setup
```
- Windows: `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass` 후 `.\install.ps1`
- Mac: `bash install.sh`

## 방법 2: 자동 설치가 안 될 때 — 개별 설치

자동 스크립트가 실패하면, 아래를 **순서대로 직접** 하세요.

### 1) Node.js 설치 (자바스크립트 실행기, 22 버전 이상)

- 공식 다운로드: **https://nodejs.org/**
  → 큰 버튼(LTS) 다운로드 → 설치 파일 실행 → 계속 "다음"
- 설치 확인 (PowerShell/터미널에서):
  ```bash
  node --version
  ```
  `v22.x.x` 처럼 나오면 성공.

### 2) GitHub Copilot CLI 설치

- 터미널/PowerShell에서:
  ```bash
  npm install -g @github/copilot
  ```
- Mac에서 권한 오류가 나면:
  ```bash
  sudo npm install -g @github/copilot
  ```
- 공식 설치 안내(다른 방법 포함): **https://docs.github.com/en/copilot/how-tos/copilot-cli/set-up-copilot-cli/install-copilot-cli**

### 3) 설치 확인

```bash
copilot --version
```
버전 번호가 나오면 성공입니다.

## 설치 후 첫 실행

> ✅ 설치 스크립트가 **작업 폴더를 이미 만들어 뒀습니다.** 위치는 항상 고정:
> - Windows: `C:\Users\내계정\copilot-workspace\invaders`
> - Mac: `~/copilot-workspace/invaders`
>
> 앞으로 Copilot 작업은 이 `copilot-workspace` 폴더에서 하면 됩니다 (상설 작업 공간).

1. 작업 폴더로 **이동** (새로 만들 필요 없음):
   - Windows(PowerShell):
     ```powershell
     cd "$env:USERPROFILE\copilot-workspace\invaders"
     ```
   - Mac(터미널):
     ```bash
     cd ~/copilot-workspace/invaders
     ```
2. **진행자가 알려준 갤러리 주소**를 등록 (copilot 켜기 **전에**):
   - Windows(PowerShell):
     ```powershell
     $env:LAB502_DASHBOARD_URL = "https://rg-hub-web-0614.azurewebsites.net"
     ```
   - Mac(터미널):
     ```bash
     export LAB502_DASHBOARD_URL="https://rg-hub-web-0614.azurewebsites.net"
     ```
   > ⚠️ 이 주소는 **새 터미널 창을 열 때마다 다시** 입력해야 합니다.
3. Copilot 켜기:
   ```bash
   copilot
   ```
4. 켜진 화면에서 `/login` 입력 → 안내대로 GitHub 로그인.

> 📁 폴더를 못 찾겠으면:
> - Windows 파일 탐색기 주소창에 `%USERPROFILE%\copilot-workspace` 입력
> - Mac Finder에서 `이동 → 폴더로 이동` → `~/copilot-workspace`

이제 [`../docs/00-시작하기.md`](../docs/00-시작하기.md)로 가세요!
