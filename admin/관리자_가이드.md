# 🛠️ 관리자(진행자) 가이드

이 문서는 **행사를 준비하는 진행자**를 위한 것입니다. 참가자가 보는 문서가 아닙니다.

진행자가 할 일은 크게 셋입니다:
1. **공용 갤러리(Community Hub)** 를 Azure에 띄운다 — 참가자들이 게임·스크린샷을 한곳에서 공유·구경
2. **참가자 환경**을 준비한다 — 자동 설치 스크립트 안내 또는 Codespaces 제공
3. 행사 후 **정지**, 다음 행사 전 **재시작** (게임은 계속 누적 보존)

---

## 📦 0. 준비물 (진행자 PC)

| 항목 | 용도 |
|------|------|
| Azure 구독 | 공용 갤러리 호스팅 |
| Azure CLI (`az`) | 배포 명령 |
| .NET 10 SDK | 갤러리 앱 빌드 |
| `git`, `zip`, `jq` | 배포 보조 (Windows면 **Azure Cloud Shell** 사용 추천) |
| LAB502 저장소 (이 핸즈온 포함) | 소스 |

> 💡 Windows에서 `zip`/`jq`가 번거로우면 **Azure Portal의 Cloud Shell(Bash)** 에서 작업하세요. 필요한 도구가 이미 다 들어 있습니다.

---

## 🌐 1. 공용 갤러리(Community Hub)를 Azure에 배포

### 왜 이렇게 하나 (설계 의도)

- 참가자들이 만든 게임·스크린샷을 **한 갤러리에 누적**해서, 다음 회차 참가자도 이전 작품을 구경하며 즐기게 합니다.
- 비용을 줄이려고 **Azure SQL 대신 SQLite 파일 DB**를 쓰고, 게임·스크린샷은 **Azure Blob**에 저장합니다.
- 행사 사이엔 앱만 **정지**하면 비용이 거의 안 들고, 데이터(게임)는 그대로 남습니다.

### 1-1. SQLite 패치 적용

원본 Community Hub는 Azure SQL 전용이라, 비용을 줄이기 위해 **SQLite 모드**를 추가하는 패치를 준비했습니다. (`admin/sqlite-patch/` 폴더)

LAB502 저장소를 clone한 폴더에서:
```bash
git apply /경로/admin/sqlite-patch/sqlite-mode.patch
```
또는 `admin/sqlite-patch/new-files`, `changed-files` 를 해당 위치에 복사. 자세한 설명·검증법은 [`sqlite-patch/README_적용_및_운영가이드.md`](sqlite-patch/README_적용_및_운영가이드.md) 참고.

> ⚠️ 빌드 확인 필수: `cd src/community-hub && dotnet build CommunityHub.slnx`

### 1-2. 배포 실행

[`scripts/01-배포.sh`](scripts/01-배포.sh) 의 맨 위 변수(리소스 그룹·지역·이름)를 본인 값으로 바꾼 뒤:

```bash
az login
bash admin/scripts/01-배포.sh
```

끝나면 **갤러리 주소**(`https://<이름>.azurewebsites.net`)가 출력됩니다. 이 주소를 참가자에게 안내하세요.

확인:
- `https://<주소>/activity` — 실시간 활동 보드
- `https://<주소>/gallery` — 게임 갤러리
- 진행자 본인이 게임 하나 올려보고 갤러리에 뜨는지 미리 테스트

---

## 🔁 2. 행사 사이 정지 / 재시작 (비용 절감 + 게임 보존)

게임·스크린샷(Blob)과 SQLite 파일(`/home`)은 **앱을 정지해도 보존**됩니다. 컴퓨팅만 끄면 됩니다.

- 행사 종료 후:  `bash admin/scripts/02-정지.sh`
- 다음 행사 전:  `bash admin/scripts/03-재시작.sh`

> ⚠️ **삭제(delete)는 절대 금지.** 앱이나 스토리지를 지우면 누적된 게임이 사라집니다. 목적과 정반대예요. 정지(stop)만 하세요.

> 💰 정지 중에도 약간의 비용(스토리지·App Service 요금제)은 남습니다. 더 줄이려면 요금제를 낮추거나, 정 안 쓰면 다음 회차까지 정지 상태로 두면 됩니다.

## 💻 3. 참가자 환경 준비 — 두 가지 방식

### 방식 A: 참가자 PC에 직접 설치 (기본)

참가자는 `setup/` 폴더의 자동 설치 스크립트를 실행합니다.
- Windows: `setup/windows-설치.ps1`
- Mac: `setup/mac-설치.sh`
- 자동 설치 실패 시 개별 설치 링크: `setup/설치_가이드.md`

진행자는 **갤러리 주소**와 아래 한 줄을 슬라이드/유인물에 크게 띄워주세요:
```
(Windows) $env:LAB502_DASHBOARD_URL = "https://rg-hub-web-0614.azurewebsites.net"
(Mac)     export LAB502_DASHBOARD_URL="https://rg-hub-web-0614.azurewebsites.net"
```

설치되는 것: **Node.js 22 + GitHub Copilot CLI** (`npm install -g @github/copilot`). 참가자별 **GitHub Copilot 구독** 필요.

### 방식 B: Codespaces (브라우저만으로 실습, 설치 0)

참가자 PC에 아무것도 안 깔고 브라우저만으로 실습하게 하려면 Codespaces를 씁니다.

**준비 (진행자 1회):**
1. LAB502 저장소(또는 이 핸즈온 저장소)를 **본인 계정으로 포크**.
2. `admin/.devcontainer/` 의 `devcontainer.json` 과 `setup.sh` 를 **저장소 루트의 `.devcontainer/`** 로 커밋.
3. `admin/.github/agents/cs-web-screenshotter.agent.md` 를 **저장소 루트의 `.github/agents/`** 로 커밋.

→ 이러면 참가자가 그 저장소에서 **Code → Codespaces → Create codespace** 할 때 .NET·Node·Copilot CLI·Playwright가 자동 설치됩니다.

**참가자 실습 순서:**
1. Codespace 생성 (1~2분 자동 세팅)
2. 터미널에서 `export LAB502_DASHBOARD_URL="https://rg-hub-web-0614.azurewebsites.net"`
3. `copilot` → `/login`
4. `docs/` 모듈 진행

**모듈 04 (스크린샷) 차이점:** Codespaces엔 화면이 없어 원본 `web-screenshotter`(Edge 창)가 안 됩니다. 대신 **`cs-web-screenshotter`** (headless)를 쓰게 했습니다. 참가자는 모듈 04에서 `/agent` 로 `cs-web-screenshotter` 를 고르면 됩니다. (캡처·공유는 되고, 브라우저 창 뜨는 시각적 재미만 없음 → 진행자가 로컬에서 한 번 시연 권장.)

> 💰 Codespaces는 사용 시간만큼 과금/무료한도 차감됩니다. 인원 × 시간을 가늠해 조직 정책·한도를 미리 확인하세요.

### 방식 A vs B 비교

| 항목 | A: 로컬 설치 | B: Codespaces |
|------|-------------|---------------|
| 참가자 준비물 | Node+CLI 설치 | 브라우저만 |
| 환경 편차 사고 | 흔함 | 거의 없음 |
| 모듈 04 브라우저 창 | 뜸 (풀 경험) | 안 뜸 (headless) |
| 비용 | 무료(각자 PC) | Codespaces 시간 |

→ **둘 다 준비**해두고 당일 참가자 환경에 따라 고르는 걸 권합니다.

## ✅ 4. 진행자 당일 체크리스트

- [ ] SQLite 패치 적용 후 `dotnet build` 성공

- [ ] 진행자 게임 1개 업로드 → 갤러리 표시 확인
- [ ] (보존 검증) `02-정지.sh` → `03-재시작.sh` 후에도 그 게임이 남아있는지 확인
- [ ] 참가자 안내자료에 **갤러리 주소** + `LAB502_DASHBOARD_URL` 한 줄 명시
- [ ] (방식 B면) 포크 저장소에 `.devcontainer/` + `.github/agents/` 커밋 완료
- [ ] 참가자 GitHub Copilot 구독 안내
- [ ] 모듈 04 브라우저 자동화를 진행자 화면에서 미리 시연 준비

```
admin/
├── 관리자_가이드.md            ← 지금 이 문서
├── scripts/
│   ├── 01-배포.sh              ← Azure 배포 (SQLite 모드)
│   ├── 02-정지.sh              ← 행사 후 정지

└── .github/agents/
    └── cs-web-screenshotter.agent.md   ← Codespaces용 headless 캡처 도우미
```
